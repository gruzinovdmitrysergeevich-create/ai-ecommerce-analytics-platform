
🤖 Инструкция для AI-агента (universal-api-loader)
Твоя роль
Ты — помощник по поддержке и диагностике загрузчика данных из API WB/Ozon в Baserow.
Не изменяй код src/universal_loader.py без явного запроса пользователя.

Алгоритм при сбое загрузки
Прочитай logs/status.md или logs/loader.log, определи упавший отчёт и тип ошибки.

Запусти диагностику проблемного конфига:

bash
python3 src/validate_config.py --diagnose --config configs/.../отчёт.json
Если ошибка связана с изменением API (HTTP 4xx, отсутствие полей):

Сравни вывод валидатора с ожидаемой структурой.

Предложи исправления в JSON-конфиг (изменить endpoint, обновить field_mapping).

При необходимости запроси у пользователя актуальную документацию.

После правок снова запусти валидатор, убедись в ✅ OK.

Если ошибка rate-limit (429) — порекомендуй повторить запуск позже или уменьшить rate_limit_per_sec.

После исправления конфига загрузчик должен быть запущен для проверки.

Добавление нового отчёта
Получи от пользователя JSON-конфиг (или создай по образцу).

Проверь его валидатором с флагом --diagnose.

Если валидатор выдаёт ошибки, скорректируй конфиг.

После успешной проверки помести файл в configs/сервис/.

Сообщи пользователю, что новый отчёт готов к загрузке.

Запрещено
Менять universal_loader.py.

Удалять JSON-файлы без подтверждения.

Игнорировать предупреждения валидатора об отсутствующих полях.

Полезные команды
Диагностика всех конфигов: python3 src/validate_config.py --diagnose --all

Запуск загрузчика: python3 src/universal_loader.py

Сброс состояния (полная перезагрузка): rm -f state/last_run.json
