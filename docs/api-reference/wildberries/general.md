# Wildberries API – общие принципы (FBS)

## Авторизация
- Все запросы требуют заголовок `Authorization: Bearer <токен>`.
- Токены создаются в личном кабинете продавца.
- Для разных категорий методов нужны разные категории токенов (например, `Контент`, `Маркетплейс`, `Аналитика`).

## Форматы дат
- Большинство методов используют RFC3339: `YYYY-MM-DD` или `YYYY-MM-DDTHH:MM:SSZ`.
- В некоторых старых методах (например, `/api/v1/supplier/sales`) время передаётся в часовом поясе Москвы (UTC+3).

## Пагинация
- В разных группах методов пагинация реализована по-разному:
  - `limit` + `offset`
  - `take` + `skip`
  - `cursor` (например, в карточках товаров)
  - последовательная подгрузка по `lastChangeDate` или `rrdid` (в отчётах)

## Лимиты запросов
- У каждого метода свои лимиты (указаны в описании).
- Общий принцип: не более 300 запросов в минуту на аккаунт для большинства операционных методов.
- При превышении возвращается `429 Too Many Requests`.

## Особенности FBS
- **Сборочные задания** проходят цепочку статусов: `new` → `confirm` (добавлено в поставку) → `complete` (поставка передана в доставку).
- **Поставки** объединяют задания для отгрузки на склад WB.
- **Метаданные** (коды маркировки) нужно добавлять до перевода в доставку, иначе поставка не закроется.

## Обработка ошибок
- Тело ошибки обычно содержит поля `title`, `detail`, `requestId`, `origin`.
- Коды HTTP стандартные: 400, 401, 403, 404, 409, 429, 500.
