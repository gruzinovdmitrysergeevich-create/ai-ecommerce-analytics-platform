# Ozon Seller API – общие принципы (FBO)

## Авторизация
- `Client-Id` и `Api-Key` в заголовках.
- Ключи действуют 6 месяцев, после чего нужно сгенерировать новый.
- Можно ограничить доступ по IP.

## Форматы дат
- Все даты в UTC, формат RFC3339: `YYYY-MM-DDTHH:MM:SSZ`.

## Пагинация
- В большинстве методов: `page` + `page_size` (до 1000).
- В аналитике: `offset` + `limit`.
- В некоторых методах (например, транзакции) – `cursor`.

## Лимиты запросов
- Общий лимит: 50 запросов в секунду на один Client ID.
- У отдельных методов есть свои ограничения (например, обновление цен – 10 раз в час на товар).

## Особенности FBO
- Остатки на складах Ozon обновляются автоматически, управлять ими через API нельзя (только чтение).
- Для управления поставками используются черновики (`draft`) и заявки (`supply-order`).

## Асинхронные отчёты
- Многие отчёты (финансы, аналитика) формируются асинхронно.
- Сначала запрос возвращает `task_id` или `code`, затем нужно периодически проверять статус и скачивать файл.

## Обработка ошибок
- Тело ошибки содержит `code`, `message`, `details`.
- Коды HTTP стандартные.
