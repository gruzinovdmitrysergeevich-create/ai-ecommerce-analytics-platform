# Агрегатор данных для маркетплейсов (WB, Ozon) и финансов

## Цель
Создавать компактные агрегированные таблицы в Baserow на основе сырых данных из universal-api-loader (маркетплейсы) и finance-loader (финансы). Агрегаты служат источником для Metabase и AI-аналитики.

## Текущее состояние
✅ Реализован и работает. Три агрегированные таблицы создаются/обновляются автоматически:
- `wb_aggregated` — логистика, штрафы, комиссия, заказы по дням
- `ozon_aggregated` — транзакции, продажи, доставка по дням
- `finance_aggregated` — выписка по счёту (дебет/кредит/баланс) по дням

## Запуск
```bash
cd ./aggregator
python3 src/aggregator.py
```

## Структура
```
aggregator/
├── AGENTS.md              ← эта инструкция
├── README.md              ← документация
├── requirements.txt       ← зависимости
├── src/
│   └── aggregator.py      ← скрипт (запускать отсюда)
├── configs/
│   └── aggregation_rules.json  ← правила агрегации
├── logs/                  ← логи (создаются при запуске)
└── memory-bank/           ← проектная документация
```

## Как добавить новую метрику
1. Отредактировать `configs/aggregation_rules.json`
2. Добавить метрику в существующее правило или создать новое
3. Запустить `python3 src/aggregator.py`

## Как перезапустить агрегацию для исторических данных
```bash
cd ./aggregator
python3 src/aggregator.py
```
Скрипт автоматически очистит целевые таблицы и перезаполнит их.

## Важные правила
- **НЕ хардкодить** названия таблиц или сервисов — всё через API Baserow.
- **Логировать** каждый шаг в `logs/run_*.log`.
- Ключи брать из `~/my-ai-stack/analytics/.env` (BASEROW_EMAIL, BASEROW_PASSWORD).
- Структуру сырых таблиц только читать, не изменять.
