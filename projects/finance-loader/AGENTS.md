# AGENTS.md — инструкция для Hermes: finance-loader

## Что делает скрипт
`src/finance_loader.py` — универсальный загрузчик финансовых данных в Baserow.
Читает Excel/CSV/Google Sheets из настроенных источников, автоматически определяет
типы колонок (число/дата/текст), создаёт таблицы и поля в Baserow,
загружает данные с контролем дубликатов по MD5-хешу.

## Запуск
```bash
cd /mnt/3E7ADD3C7ADCF19F/ai-projects/finance-loader
python3 src/finance_loader.py
```

## Как работает загрузчик

### Автоопределение типов
Скрипт сам смотрит на данные в каждой колонке и определяет тип:
- **integer** — целые числа
- **number** — дробные (в т.ч. с русской запятой: `285,0` → 285.0)
- **date** — даты (российские и стандартные форматы)
- **text** — всё остальное

Колонки с `руб`, `₽`, `RUB`, `%` никогда не считаются датами.

### Автоопределение заголовков
Скрипт распознаёт три ситуации:
1. **Обычная таблица** — первая строка = заголовки. Стандартно.
2. **Мультизаголовок** (ozon_internal) — первая строка = заголовок отчёта, вторая = колонки.
3. **ВТБ-выписка** — 6 служебных строк, заголовки в строке 7.
4. **Управленка** — 2 строки заголовка, колонки в строке 3.

Всё определяется автоматически по содержимому, без привязки к конкретным файлам.

### Дедупликация
По MD5-хешу от полей, указанных в `hash_fields` конфига.
- При первом запуске: загружаются все уникальные строки.
- При повторном: только новые (хеш уже есть → пропускается).
- Файлы НЕ удаляются после загрузки (`delete_after_upload: false`).

### Инкрементальная загрузка (только Управленка)
`incremental: true` — загружаются только новые хеши.
Остальные источники — `false` (каждый раз проверяются все строки).

## Что делать при изменениях

### Появился новый файл в локальной папке
Ничего не делать. При следующем запуске скрипт сам его подхватит.
Новые колонки автоматически добавятся в таблицу Baserow.

### Изменилась структура файла (новые колонки)
Автоматически. Новые колонки создадутся в Baserow.
Старые колонки продолжат работать, в новых строках для них будет `null`.

### Исчезли файлы из папки
Ничего не происходит. Скрипт просто не находит файлов → нечего загружать.

### Изменилось название файла/папки на Google Диске
Поправить `path` и/или `file_filter` в соответствующем JSON-конфиге.

### Новый источник (новая папка или Google Drive)
1. Скопировать любой JSON из `configs/`
2. Поправить: `path`, `database`, `table_name`, `hash_fields`, `file_filter`
3. Запустить скрипт — таблица и поля создадутся автоматически

### Google-авторизация (сервисный аккаунт)
Ключ лежит в `~/my-ai-stack/analytics/service_account.json`.
Не протухает. Если перестал работать — проверить что сервисному аккаунту
выдан доступ к нужным папкам на Google Диске (роль «Читатель»).
Email: `finance-loader-sa@finance-loader.iam.gserviceaccount.com`

## Формат конфига (JSON)
```json
{
  "source_type": "local или google_drive",
  "path": "путь к папке или файлу",
  "database": "имя рабочего пространства Baserow",
  "table_name": "имя таблицы",
  "hash_fields": ["ключевые", "поля", "для", "дедупликации"],
  "defaults": {"Маркетплейс": "WB"},
  "delete_after_upload": false,
  "incremental": false,
  "file_filter": "*.xlsx или точное имя",
  "sheet_name": null,
  "skip_sheets": ["листы", "которые", "пропустить"]
}
```

- `delete_after_upload: false` — файлы НЕ удаляются. Безопасно.
- `incremental: true` — только для управленки (файл не удаляется, данные дополняются).
- `sheet_name: null` — читать все листы (для ВТБ и управленки).
- `file_filter` — если папка: маска файлов (`*.xlsx`) или точное имя.
- `skip_sheets` — имена листов, которые нужно пропустить (префиксное совпадение).
