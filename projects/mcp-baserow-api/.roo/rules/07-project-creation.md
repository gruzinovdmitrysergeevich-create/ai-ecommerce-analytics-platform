# Создание нового проекта (общий процесс для Roo Code, Kilo Code, OpenCode)

## 1. Исходные данные
- Шаблон проекта находится в `~/my-ai-stack/templates/project-template/`.
- Все проекты создаются в `/mnt/3E7ADD3C7ADCF19F/ai-projects/`.
- Глобальные правила уже загружены, инфраструктура известна.
- Глобальные правила для Kilo Code доступны через симлинк `~/.kilocode/rules` → `~/.roo/rules`.

## 2. Этапы создания проекта

### 2.1. Обсуждение архитектуры и наличие projectbrief.md
Когда ты открываешь новую папку в VS Code и начинаешь диалог с агентом, **агент должен**:
- Проверить, существует ли файл `memory-bank/projectbrief.md` в корне проекта.
- **Если файл существует** – прочитать его, использовать как утверждённую архитектуру.
- **Если файл не существует** – запросить у тебя детали (цель, сервисы, отчёты, структуру) и создать `memory-bank/projectbrief.md` на основе обсуждения.
- **После утверждения архитектуры** (файл создан или уже есть) ты даёшь команду **«создай структуру проекта»**.

### 2.2. Создание структуры проекта
Агент выполняет следующие действия:

1. **Создаёт папку проекта** (если ещё не создана) по пути `/mnt/3E7ADD3C7ADCF19F/ai-projects/<project-name>/`.
2. **Копирует шаблон** из `~/my-ai-stack/templates/project-template/` в папку проекта, **адаптируя содержимое под конкретный проект**:
   - заменяет `{{PROJECT_NAME}}` на фактическое имя;
   - обновляет `memory-bank/projectbrief.md` (если уже есть – оставляет без изменений или сливает);
   - обновляет `memory-bank/techContext.md`, `memory-bank/activeContext.md` согласно утверждённой архитектуре;
   - наполняет локальные правила (`.roo/rules/`) спецификой проекта (API, таблицы Baserow, MCP).
3. **Создаёт локальные правила для Kilo Code**:
   - Создаёт папку `.kilocode` в корне проекта, если её нет.
   - **Копирует** все файлы из `.roo/rules/` в `.kilocode/rules/` (без симлинка, чтобы Kilo Code отображал правила в интерфейсе).
4. **Настраивает локальные правила для OpenCode (плагин opencode-rules)**:
   - Создаёт симлинк `.opencode/rules` → `.roo/rules` внутри проекта, чтобы плагин мог читать локальные правила.
   - Команда: `ln -sf .roo/rules .opencode/rules`
5. **Сохраняет изменения** и сообщает: «Структура проекта готова. Можно приступать к реализации.»

### 2.3. Доступ к шаблону

Шаблон проекта: `~/my-ai-stack/templates/project-template/`
- **Roo Code / Kilo Code** могут прочитать шаблон только при включённом MCP `filesystem`. Перед созданием проекта они **обязаны** запросить у пользователя временное включение этого MCP.
- **Терминальный агент (OpenCode)** имеет прямой доступ и может скопировать шаблон без разрешения.

## 3. Примечания
- Все действия должны выполняться последовательно, с проверкой успешности каждого шага.
- Если в шаблоне есть файлы с примерами кода или JSON, их нужно либо удалить, либо переименовать, либо явно адаптировать под новый проект (агент должен запросить уточнения).
- После создания структуры агент может предложить перейти к реализации первого модуля.

## 4. Пример команды для создания
Ты можешь использовать фразу:
> «Создай структуру проекта для [название] на основе утверждённой архитектуры. Используй шаблон и адаптируй под [детали].»

## 5. Примечания для терминального агента (OpenCode)
- OpenCode выполняет создание проекта автоматически (без запроса MCP), следуя тем же шагам: копирует шаблон, адаптирует, создаёт локальные правила для Roo/Kilo, настраивает memory‑bank, создаёт симлинк `.opencode/rules` → `.roo/rules`.
